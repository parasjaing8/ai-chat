# Agent Rules for Generated Projects

These rules apply to ALL code generated by AI agents for projects built on this platform.

## Deployment Environment

- **Machine**: Mac Mini, Apple Silicon, 16GB RAM, macOS — **headless** (no display, keyboard, or mouse)
- **Access**: Browser only via http://192.168.0.130:8080 or `/play/<slug>/`
- **Port 8080**: Reserved for the AI chat server — NEVER use for project servers

## Web Project Rules

### Entry Point
Always create `index.html` as the project entry point. It is served at `/play/<slug>/`.

### Asset Paths
Use **relative paths** for all assets:
```html
<!-- CORRECT -->
<script src="js/game.js"></script>
<link rel="stylesheet" href="css/style.css">

<!-- WRONG -->
<script src="/js/game.js"></script>   <!-- absolute path -->
<script src="C:/path/to/file"></script>  <!-- filesystem path -->
```

### HTML Boilerplate
Every HTML file must include:
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Project Title</title>
</head>
```

### No Local File Instructions
Never tell the user to "open a file", "double-click", "use Finder/Explorer", or reference any GUI. The machine is headless. Always provide the play URL instead:
```
http://192.168.0.130:8080/play/<project-slug>/
```

### Complete Code
Never include stubs or placeholders:
- No `// TODO: implement this`
- No `/* placeholder */`
- No stub functions
- No references to files that don't exist in the project

### Touch Controls for Games
Canvas/game projects must handle both keyboard AND touch input:
```js
// Show touch controls on touch devices
// Use @media (pointer: coarse) { .touch-controls { display: flex; } }
```

## No Additional Servers
Do NOT start Express, Flask, http-server, or any other server process. Projects are auto-served via the `/play/` route — no extra server needed.
